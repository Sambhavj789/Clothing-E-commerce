Backend Folder Structure:
backend 
 |
 src ----|
        server.js (Main Entry Point of backend)
        routes (Folder that will contain all routes files)
        controllers (Folder that will contain handler functions)
        models (Folder that will contain all database models/tables)
        config (Folder that will contain all configuration files like connection to db)
 .env (File that contains all environment varaibles, it contains varaibles which we cannot share to other users)

Backend Libraries:
mongoose (For dealing with database)
express (For making server)
dotenv (For accessing varaibles of .env file)
cors (For allowing backend to accept request from out frontend)
nodemon (For restarting server whenever code changes)
bcrypt (For password hashing)
jsonwebtoken (For generating token)
cookie-parser (For handling cookie related work)
multer (Used to manage file uploads)

Command:
npm i express mongoose cors dotenv nodemon bcrypt jsonwebtoken cookie-parser multer

Routes:
autRoutes: All routes related to Authentication

POST: /register 
POST: /login
GET: /me 
GET: /logout

categoryRoutes: All routes related to manageing category
GET: / (Get all category)
POST: / (Add new category)
PUT: / (Update old category, or add subcategory inside it)
Delete: / (Delete category)

productRoutes: All routes related to products:
GET: /all (Get all products)
GET: /:productId : (Get single product)
POST: /: (Add a single product)
PUT: /: (Update product)
DELETE: / : (Delete Product)


orderRoutes: All routes related to order 
GET: /all: Get all orders of a customer/user. (Only access to admin)
GET: /user-orders: Get all orders for current login users. (User must be login)
POST: /create : Used to create order. (User must be login)
PUT: /update-status: Used to update order status (Only access by admin)
